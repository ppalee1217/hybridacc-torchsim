HACC Core Controller 硬體架構規格書

本頁將 Core.md 的內容完整鏡射到 HTML，並額外補上多張內嵌 SVG micro-architecture block diagrams，讓 reviewer、RTL 工程師與系統整合工程師能直接對照控制器的資料路徑、命令路徑、載入路徑與 block execution 路徑。

設計核心

極簡 control core、descriptor SRAM、block expander、cluster command engine、DMA engine 與多 cluster replicated frontends。

載入路徑

`.hacc.core/.job/.block/.dma/.agu/.patch` 主要進 local memory；`.hacc.pe/.scan` 轉成 NoC AHB command stream；runtime tensor data 才走 AXI4-Lite SPM path。

契約基準

目前 cluster 介面以 ComputeCluster 實作為準：AHB `0x0000~0x00FF` SPM config、`0x1000~0x1FFF` HDDU MMIO、`0x2000` NoC command；AXI4-Lite data path 對應 SPM 線性視窗。

Top-level Controller Diagram

整體控制器分成 host/loader、local memory、descriptor execution、DMA、cluster command 與 interrupt/perf 六大區塊。

Section Type to Hardware Route

這張圖把 `.hacc.*` section 與實際硬體路徑關係視覺化，避免 reviewer 把 control/program payload 跟 runtime tensor payload 混在一起。

Section Loader Micro-architecture

section loader 必須同時處理 local write、SPM AXI payload 與 NoC AHB command 轉換三種不同目的地。

Block Execution Pipeline

這張圖把 Job Manager、Block Fetch、Block Expander、Wave Scheduler 與 cluster execution 的 pipeline 串起來。

Block Expander Internal Diagram

Block Expander 是 block-first 架構的核心；這張圖把 loop 展開、公式運算、patch 覆寫與 emit FIFO 切得更細。

Wave Scheduler State Diagram

Wave Scheduler 的核心不是大量 script 解碼，而是固定狀態機：prefetch、apply、start、wait、drain、commit。

DMA Engine Data Path

DMA engine 需要同時理解 formula mode 與 address-list mode，並在 DRAM AXI burst 與 cluster AXI4-Lite single-beat 間做節奏轉換。

Cluster Command Engine Routing

cluster command engine 必須明確分成 HDDU MMIO passthrough 與 NoC sideband 兩條 AHB 路徑，不能再退回舊版抽象 command CSR 模型。

IRQ / Completion / Timeout Feedback

完成與錯誤回授是 scheduler 可用性的核心，必須先 sticky capture，再用 mask/timeout 邏輯整合。

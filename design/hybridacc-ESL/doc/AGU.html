Timing Notes

  • Loop 前進語意：只有在輸出端 gen_valid && gen_ready 成功握手後，該筆 descriptor 才算被消耗。
  • next_idx/all_done 在當拍計算，idx_reg 於下一拍註冊更新，避免 carry chain 直接壓在輸出控制路徑。
  • 位址路徑使用 n0_s0 -> n0_s1 -> n0_out 三級子管線；tag 乘法在 s0->s1 完成。
  • 當下游背壓（gen_ready=0）時，pipeline 以 valid/ready 反壓保留資料，不丟失 descriptor。
  • 最後一筆（last=1）完成握手後產生 done pulse，FSM 進入 DONE 再回 IDLE。

  • Backpressure：若 T3 時 gen_ready=0，s2_valid 保持為 1，並沿 valid/ready 反壓到前級。
  • Loop update：next_idx/all_done 先算出，idx_reg 於下一拍註冊更新。
  • Startup bypass：當 n0 子管線為空時，可直接由 S1 前饋到輸出 pack，降低首包啟動延遲。
